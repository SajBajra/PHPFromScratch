Notes app (per-user CRUD)

I built a small notes app using the same php_learn database. There is a notes_app_notes table with id, user_id, title, body (TEXT), and created_at. 28-notes-list.php is protected with require-auth; it creates the table if needed, then selects all notes for the logged-in user (by auth_user_id) ordered by created_at and shows them with a link to add a new note. 28-notes-add.php shows a form for title and body; on POST it uses csrf_token()/csrf_verify(), validates title and body (required, title max 255 chars), inserts a row with the current user's id, and redirects back to the list with a flash message. 28-notes-view.php loads a single note by id and user_id and shows title, date, body, and links to edit/delete. 28-notes-edit.php loads the note for the current user, shows a pre-filled form, and on POST uses CSRF and the same validation before updating the row and redirecting back to the view page with a flash. 28-notes-delete-confirm.php loads the note for the current user and shows a confirmation form that posts (with a CSRF token) to 28-notes-delete.php, which checks CSRF and deletes the note only when id and user_id match. The header now shows a Notes link for logged-in users that goes to 28-notes-list.php, so each user has their own private list of notes.
