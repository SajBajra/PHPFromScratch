Today I started learning how PHP works together with HTML forms using the GET method. I created a simple page that shows a text box for my name and a submit button. The form uses `method="get"` and when I submit it, the browser sends the value in the query string, which PHP reads from the `$_GET` superglobal array.

In my example, I read the name like this: `$name = $_GET['name'] ?? '';`. If the name is not set yet, I use an empty string as a default. When the name is present, I show a message such as `Hello, Saj!` on the page. I also used `htmlspecialchars` when I print the name, so that any special characters are safely escaped before they are rendered in the HTML.

After that, I built several POST-based examples. First I converted the simple form to use `method="post"` and read values from `$_POST` instead of `$_GET`. Then I added basic validation, so that if the name is empty I show an error message and keep the submitted value in the input. I also created a multi-field form (name, email, age, message) with simple checks like required fields, valid email format, and numeric age. I then split a form and its processing logic into two separate files: one file only shows the form and another file receives the POST data, validates it, and either shows errors or a thank-you/summary page. Finally, I added two small pages that use `$_SESSION` to remember the user’s name across requests: one page sets the name in the session, and another page reads it and greets the user.
