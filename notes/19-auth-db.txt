Auth with database (register, login, logout)

I added real authentication using the database. The users table has id, email (unique), password_hash, and created_at. 26-register.php creates the table if needed, validates email and password (length, confirm match), checks for duplicate email, hashes the password with password_hash(), inserts the user, and redirects to login with a success message. 26-login.php looks up the user by email and checks the password with password_verify(); on success it sets auth_logged_in, auth_user_id, and auth_email in the session and redirects to 26-dashboard.php or back to the originally requested page if require-auth had stored it in auth_redirect_to. 26-dashboard.php only loads if auth_logged_in is set, otherwise redirects to login. 26-logout.php clears the auth session keys and redirects to login. This replaces the old hard-coded login (10-*) with database-backed auth and proper password hashing.

I also added Register and Log in links to the shared header (includes/header.php); when logged in the header shows Dashboard, Profile, and Log out instead. The blog add post page (27-blog-add.php) is protected: only logged-in users can open it (others are redirected to login). On the blog list (27-blog-list.php) the "Add post" link is shown only when logged in; otherwise it shows "Log in to add a post". I also added includes/require-auth.php: an include that starts the session if needed, remembers the requested URL in auth_redirect_to, and redirects to 26-login.php if the user is not logged in. The dashboard, blog add page, profile page, and other protected pages now require that file instead of repeating the same check, so the "require login" (and redirect-back) logic lives in one place.
