Auth with database (register, login, logout)

I added real authentication using the database. The users table has id, email (unique), password_hash, and created_at. 26-register.php creates the table if needed, validates email and password (length, confirm match), checks for duplicate email, hashes the password with password_hash(), inserts the user, and redirects to login with a success message. 26-login.php looks up the user by email and checks the password with password_verify(); on success it sets auth_logged_in, auth_user_id, and auth_email in the session and redirects to 26-dashboard.php. 26-dashboard.php only loads if auth_logged_in is set, otherwise redirects to login. 26-logout.php clears the auth session keys and redirects to login. This replaces the old hard-coded login (10-*) with database-backed auth and proper password hashing.
