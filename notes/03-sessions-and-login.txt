Today I practised using PHP sessions to build a very simple login system. I created a login form that posts a username and password to a separate processing script. The processing script compares the values against hard-coded credentials, and if they match it stores `logged_in = true` and the username in `$_SESSION`. There is also a small "dashboard" page that checks the `logged_in` flag before showing its content, and if the user is not logged in it redirects back to the login form. Finally, I added a logout script that clears the session and sends the user back to the login page. This flow shows how sessions can keep track of whether a user is logged in across multiple requests.
